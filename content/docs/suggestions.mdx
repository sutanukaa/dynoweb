The Suggestions page is where DynoWeb turns store data into prioritized recommendations.

It combines quick theme checks, behavior-based analysis, previews, and draft-safe actions so users can move from insight to improvement in one place.

<img src="/suggestionsOverview.png" alt="Suggestions page overview" />

## What It Does

This page has two starting points.

Quick Store Check looks through the live theme for easy wins and works even before the store has traffic.

Traffic Analysis uses real visitor behavior to find higher-confidence opportunities based on what shoppers are actually doing.

<img src="/storechecks.png" alt="Suggestions page quick store check and traffic analysis options" />

Once suggestions are available, they are grouped into tiers like Quick Wins, Strategic, Ambitious, and Needs Triage so users can decide what to tackle first.

Each suggestion card includes the recommendation, page context, evidence, important signals, and actions like Open Preview, Matching Replays, Apply to Draft for supported changes, Dismiss, or Revert Draft when already applied.

<img src="/suggestion1.png" alt="Suggestions page prioritized recommendation cards" />

<img src="/suggestion2.png" alt="Suggestions page recommendation preview and actions" />

The lower part of the page helps users manage the suggestion workflow.

Analysis Runs shows recent engine jobs and their results, while Preview Candidates shows unpublished candidate changes that passed internal validation and can be filtered by target, modification type, source, and sort order.

<img src="/suggestionss.png" alt="Suggestions page analysis runs and preview candidates" />

## How To Use It Best

The best flow is to start with Quick Store Check right away, especially for a newer store, because it does not depend on traffic.

After the store has enough visitor activity, users should run Traffic Analysis to generate evidence-backed suggestions based on real behavior.

From there, users should review the highest-priority suggestions first, open previews before making decisions, and apply only supported changes to a draft theme.

This page works best when users treat it as a decision queue: check the priority, review the evidence, preview the change, apply it safely to draft, and then validate results later through Experiments.

One important note for the guide: not every suggestion is one-click auto-applicable.

Some can be applied directly to a draft theme, while others are manual-review only and should be treated as guided recommendations.

## Good Help-Guide Positioning

"Suggestions is where DynoWeb turns theme checks and visitor behavior into prioritized recommendations, helping users preview fixes, apply supported changes safely to a draft theme, and track improvement opportunities over time."
